🔧 Como Funciona o JWT

1. Login
POST /api/auth/login Content-Type: application/json { "email": "admin@livora.com", "password": "sua-senha" } Resposta: { "success": true, "message": "Login realizado com sucesso.", "user": { ... }, "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." }
2. Usar Token
GET /api/users Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... Resposta: [ { "id": 1, "email": "...", ... } ]
3. Token Expirado
GET /api/users Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... Resposta (Error): 401 Unauthorized "Token expirado"

❓ Troubleshooting

❌ 401 Unauthorized
Causa: Token ausente, inválido ou expirado
Solução: Faça login novamente para obter um novo token
❌ 403 Forbidden
Causa: Token válido mas sem permissão para acessar o recurso
Solução: Verifique se seu usuário tem a role necessária (ex: Admin)
❌ Token não aparece no Swagger
Causa: JavaScript não foi carregado
Solução: Abra o console (F12) e verifique se há erros
ℹ️ Ver logs no console
Pressione F12 para abrir o DevTools e veja os logs do JWT Helper:
[Swagger JWT Helper]

💡 Dica: O token é válido por 5 minutos. Após expirar, faça login novamente.
